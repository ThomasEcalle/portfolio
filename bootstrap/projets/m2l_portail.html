M2L : Le portail utilisateur

Applcation Web codée en python

Contexte Solution proposée Structure de l'application Conclusion Compétences validées

Le contexte

La M2L (Maison des Ligues de Lauraine) est une association regroupant l'ensemble des ligues sportives de Lauraine. Monsieur Lefut est l'administrateur de l'actuelle application web de la M2L, application dont l'objectif est de pouvoir recenser l'ensemble des informations sur les membres de la M2L.

Problématique

Cette application rudimentaire n’est pas accessible aux utilisateurs, et est rédhibitoire sur le plan de la sécurité. De plus, il perd depuis des années un temps précieux à changer les adresses e-mail lui-même à la demande des utilisateurs. Son grand rêve serait qu’ils puissent changer eux-mêmes leur adresse e-mail et choisir leur mot de passe. Monsieur Lefut a décidé de faire évoluer cette application en recourant à une équipe de prestataires.

Objectifs

  • Mettre en place une page de connexion utilisateurs qui pourront alors s'inscrire et changer eux-même leurs informations.
  • L'administrateur doit pouvoir modifier les informations de n'importe quel utilisateur.

Contraintes éventuelles

Application exclusivement codée en langage Python

La solution proposée

Côté utilisateur

Afin de répondre aux attentes du client, nous avons crée une application Web permettant à un utilisateur de :

  • S'inscrire sur le portail et d'y modifier ses informations
  • Faire une demande de regeneration de mot de passe automatique en cas d'oublis de celui-ci
  • Mise en place d'une documentation utilisateur accessible sur l'application

Côté administrateur

En tant qu'administrateur, monsieur Lefut aura de nombreuses possibilités :

  • Possibilité d'inscrire un nouvel utilisateur, un mot de passe aléatoire est alors créé et envoyer directement par Mail à l'utilisateur
  • Droit de modifications et suppressions sur n'importe quel utilisateur
  • Possibilité de nommer de nouveaux administrateurs

Sécurité

L'une des exigence principales du projet était l'augmentation de la sécurité de l'application, un certains nombres de mesures ont été mises en place :

  • Mise en place de connexions sécurisée côté serveur et côté utilisateur
  • Protection contre les injections SQL grâce à l'utilisations de REGEX
  • Hashage des mots de passe utilisateurs dans la base de données

Les fonctions d'affichage

Le mot "print" placé au début du nom de la fonction précise que celle-ci est une fontion d'affichage. Son rôle sera alors d'afficher un certain texte/formulaire en fonction de celle-ci. Un exemple ici avec la fonction print_inscription

Les fonctions de traitement de données

Il s'agit des fonctions de traitements de connées dont le rôle est de récuperer les données transmises via les formulaire, de vérifier leur conformité et, le cas échéant, d'afficher les erreurs. Exemple avec la fonction verification_formulaire_ajout

La conformité des données envoyées par l'utilisateur est garantie par l'utilisation de Regex qui emêchent ce dernier de placer des caractères dangereux (injections sql) dans le formulaire. Exemple de test avec les regex :

Et voici les différets pattern utilisés pour les Regex :

Conclusion

Attentes

  • Création d'une page de connexion Utilisateur
  • Renforcement de la sécurité

Solution proposée

  • Page de connexion utilisarteur entièrement sécurisée
  • Mise en place de Regex anti-injections sql
  • Mise en place de fonction d'envoie de Mail automatique fonctionnel (pour les inscriptions, les ajouts et les mots d epass eoubliés)
  • Mise en placa d'une option de mot de passe oublié
  • Possibilité pour l'administrateur de supprimer/modifier/passer administrateur un utilisateur
  • Mots de passe sécurisés dans la base de données (hashage MD5)
  • Mise en place d'une documentation utilisateur accessible en ligne (onglet "Aide")
